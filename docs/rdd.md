## OmapiとNFCパススルー開発用 要件定義

### 1. プロジェクト概要
Omapi (Open Mobile API) を利用し、NFCを経由してセキュアエレメントやSIMカードへ非接触アクセスを可能にするパススルー機能の開発。

### 2. 想定ユースケース
- Androidデバイスからセキュアエレメント上のアプレットにNFC経由でアクセス
- NFCタグやICカードのデータをセキュアエレメントを介して読み書き
- 物理的な電極接続をNFC通信に置き換え、ワイヤレスでのデバッグ・開発

### 3. 必要な機能一覧
- ObserveモードでのAID監視・ルーティング
  - すべてのAIDをOmapiに転送試行
  - 拒否された場合は引き続きObserveモードで監視継続
  - 許可されたAID検出時にOmapiへルート確立
  - 以降、Observeモードを切断し通常HCEフローへ移行
- デバッグ用ロギング（ユーザー非表示）
  - NFC通信パケット内容の詳細ログ
  - AIDルーティング結果の記録
  - セキュアエレメントとのコマンドやり取りの要約ログ
- NFC通信の確立と維持
- Omapi APIを介したセキュアエレメントへのコマンド送信
- パススルーの双方向通信（セキュアエレメント↔NFCインターフェース↔Android）
- ログ取得とデバッグ用インターフェース

### 4. 技術的前提
- Android OSのバージョン要件
  - Android 15（NFCパススルー実現のため必須）
- 対応するNFCハードウェア仕様（標準的なNFC A/Bモード対応）
- セキュアエレメントの種類・対応プロトコル
  - SIMカード（UICC）
  - 組み込みセキュアエレメント（eSE）
  - 各セキュアエレメントへのアクセスプロトコル（ISO 7816など）

### 5. セキュリティ要件
- Omapi側で提供されるセキュアチャネル機能を利用（アプリ側で独自認証・暗号化実装は不要）
- セキュアチャネル確立後の通信はOmapiとセキュアエレメント間で暗号化
- アプリ側での追加的なセキュリティ層（例：ログ出力時のセンシティブデータマスキング等）

### 6. 開発環境
- 開発言語：JavaまたはKotlin（動作速度・開発効率に応じて選択）
- 利用ライブラリ：標準のAndroid NFC APIを主に使用（特殊な外部NFCライブラリは想定なし）
- 開発ツール：Android Studio
- 対応APIレベル：Android 15以降

### 7. マイルストーン
- プロトタイプ作成
  - 基本的なNFCパススルー動作確認（Observeモード、AIDルーティング）
- 基本機能実装
  - セキュアエレメントへのコマンド送信、パススルー通信経路確立
  - UI実装（セキュアエレメント選択画面など）
- テストとデバッグ
  - 応答時間、通信安定性確認（目安：応答最大5秒、標準300ms程度）
  - 各種AIDルーティングシナリオの検証
- 最終版リリース
  - パフォーマンスチューニング、ドキュメント整備

### 8. アプリ動作条件・インターフェース仕様
- パススルー機能はアプリがフォアグラウンドの時のみ動作
  - Observeモードもアプリがフォアグラウンド時に限定
- Omapiが対応する複数のセキュアエレメントインターフェース選択機能
  - 例：SIMカード1、SIMカード2、eSE 1、eSE 2など
  - ユーザーが手動で選択可能なUIを提供
- バックグラウンド動作時はNFCパススルー停止
- ユーザー向けインターフェースは最小限（基本的に自動処理、詳細設定は開発者向けオプションとして用意）

以上が現時点での要件定義の全体像です。
